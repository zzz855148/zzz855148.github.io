Hexo + Butterfly 搭建个人博客

发表于2026-05-19|更新于2026-05-19|教程
|浏览量:

好的，这是一篇关于如何配置 Hexo + Butterfly 主题来搭建初始博客页面的指南博客。

引言

拥有一个属于自己的个人博客，是记录学习心得、分享知识和展示作品的好地方。Hexo 是一款快速、简洁且高效的静态博客框架，而 Butterfly 是一款设计精美、功能强大的 Hexo 主题。本文将详细介绍如何一步步地使用 Hexo 框架和 Butterfly 主题，搭建出一个漂亮的初始博客页面。

准备工作

在开始之前，你需要准备好以下工具：

  1. Node.js: Hexo 是基于 Node.js 的，因此需要先安装 Node.js (建议使用 LTS 版本)。
  2. Git: 用于代码管理，也是后续部署到 GitHub Pages 的关键工具。
  3. 代码编辑器: 如 VS Code, Sublime Text 等，用来编辑配置文件和撰写文章。
  4. GitHub 账号: 如果你打算使用 GitHub Pages 来免费托管你的博客。

    为了方便自定义，建议将主题自带的 _config.yml 文件复制一份到站点根目录，并重命名为 _config.butterfly.yml。这样可以避免直接修改主题文件，便于将来升级主题。

第三步：初步配置站点信息

现在我们需要修改站点配置文件 (_config.yml) 和主题配置文件 (_config.butterfly.yml) 来个性化你的博客。

  1. 编辑站点配置 (_config.yml)：

    • 站点基本信息： 修改 title, subtitle, description, author 等字段，填入你自己的信息。
    • 站点 URL： 修改 url 为你将来要使用的域名。如果是 GitHub Pages，则格式为 https://<你的GitHub用户名>.github.io。
    • 部署设置 (可选，但重要)： 如果计划部署到 GitHub Pages，配置 deploy 部分。 需要先安装部署插件：npm install hexo-deployer-git --save

  2. 编辑主题配置 (_config.butterfly.yml)：
    这个文件包含了 Butterfly 主题的所有可配置项。你可以根据自己的需求进行修改。以下是一些重要的初始配置点：

    • 导航菜单： 修改 menu 部分来定制顶部导航栏。
    • 个人信息 (侧边栏/首页)： 配置 avatar, social 等。
    • 首页信息 (Subtitle)： 如果想在首页显示动态打字机效果的副标题，可以配置 subtitle 部分。
    • 评论、搜索等： 这些功能可以先留空，后续再根据需要配置。

第四步：生成和部署

  1. 清理并重新生成：
    每次修改配置文件后，最好先清理缓存再生成。

  2. 本地预览：
    再次运行 hexo s 并访问 http://localhost:4000，检查各项配置是否生效，页面是否美观。

  3. 部署到 GitHub Pages：
    如果你配置了 deploy，运行以下命令即可将生成的静态文件推送到 GitHub 仓库。

    部署成功后，等待几分钟（GitHub Pages 更新需要时间），访问你配置的 url (例如 https://yourname.github.io) 就能看到你的博客上线啦！

总结

通过以上步骤，你已经成功地使用 Hexo 和 Butterfly 主题搭建了一个初始的个人博客。这只是起点，你还可以继续：

  • 撰写文章： 使用 hexo new "文章标题" 创建新文章。
  • 美化主题： 进一步深入 themes/butterfly/_config.yml 或 _config.butterfly.yml 进行更细致的配置。
  • 添加功能： 配置评论系统（如 Valine, Waline, Giscus）、搜索功能（如 Algolia）、统计代码等。
  • 自定义页面： 创建关于页 (hexo new page "about")、友链页等。

享受搭建和维护你的个人博客的过程吧！
