Translate for Zotero插件配置：连接 DeepLX 与自定义 GPT 接口

发表于2026-06-25|更新于2026-06-26|教程工具效率
|浏览量:

引言

Translate for Zotero 是 Zotero 的一款翻译插件，可以在阅读 PDF 时直接翻译选中的文本。本文记录如何为它配置 DeepLX API 和兼容 OpenAI 格式的自定义 GPT 接口，以便在 DeepLX 与 GPT 两种翻译引擎之间切换使用。

准备工作

在开始之前，你需要准备好以下工具：

  • Zotero： 确保已安装最新版本的 Zotero 客户端（本文以 Zotero 7 为例）。
  • Translate for Zotero 插件： 由 windingwind 开发的开源插件（又名 Zotero PDF Translate），可翻译 PDF、EPub、网页、条目元数据、批注与笔记，支持 20 多种翻译服务（DeepL、Google、Microsoft、OpenAI 等）。
  • DeepLX 服务： 准备一个可用的 DeepLX 服务实例（API 地址），可自建或使用公开实例。
  • GPT API Key / 服务： 获取兼容 OpenAI 格式的服务地址（Base URL）与 API Key。

第一步：安装 Translate for Zotero 插件

  1. 下载插件： 前往插件的 GitHub Releases 页面，下载最新版的 translate-for-zotero.xpi 文件。若使用 Firefox 浏览器，请右键链接选择「另存为」，避免浏览器将其当作 Firefox 扩展安装。

  2. 安装插件：

    • 打开 Zotero，进入 工具 (Tools) > 插件 (Plugins)，打开插件管理器。
    • 点击右上角的齿轮图标，选择 从文件安装插件... (Install Plugin From File...)。
    • 在弹出的窗口中选择刚才下载的 .xpi 文件并确认安装。

  3. 重启 Zotero： 安装完成后，重启 Zotero 以使插件生效。重启后即可在插件列表中看到已启用的 Translate for Zotero。

第二步：打开设置并调整通用选项

进入 编辑 (Edit) > 设置 (Settings)，在左侧列表中选择 Translate 标签，即可看到插件的全部配置项。

General 区域可按需勾选常用选项，建议开启：

  • Automatically Translate Selection： 在阅读器中划选文本后自动翻译。
  • Enable Reader Selection Pop-up： 划词时弹出翻译浮窗，并可一键 Add Translation to Note，方便把译文沉淀为笔记。

其余如 Automatically Translate Annotation（自动翻译批注）可视习惯启用。

继续向下滚动到 Service 区域，即可选择并配置具体的翻译引擎。下面分别介绍 DeepLX 与自定义 GPT 两种配置方式。

第三步：配置 DeepLX API（用于翻译）

  1. 获取 DeepLX 服务： 自行部署或获取一个可用的 DeepLX 服务地址，例如 http://your-deeplx-server:port。
  2. 选择并配置服务：
    • 在 Service 区域的 Translation Services 下拉中选择 DeepLX(API)。
    • 点击右侧的 Config 按钮，在弹出的 DeepLX(API) Config 窗口中，将 EndPoint 填写为你的 DeepLX 服务地址（通常只需填到端口，插件会自动拼接请求路径）。
    • 点击 Save 保存。Secret 一项通常无需填写，具体取决于你所使用的 DeepLX 实现。
    • 在 Translate From / To 中设置源语言与目标语言（例如 English → Chinese (S)）。

第四步：配置自定义 GPT 接口（用于 AI 翻译）

DeepLX 配置简单、响应较快；GPT 服务则可以通过自定义提示词控制译文风格。两者可以同时配置，按需切换。

  1. 选择 Custom GPT 服务：

    • 在 Translation Services 下拉中选择 Custom GPT 1（插件预留了多个自定义槽位）。
    • 在右侧 Secret 中填入你的 API Key。

  2. 填写接口详情： 点击 Config 打开 Custom GPT 1 Config 窗口，依次填写：

    • API： 服务的接口地址（官方 OpenAI 或兼容服务的 Base URL，如 https://your-custom-gpt-service.com/v1/chat/completions）。
    • Model： 使用的模型名称（如 gpt-4o、gpt-4、gpt-3.5-turbo 等）。
    • Temp： 采样温度，翻译场景建议调低（如 0.3）以保证稳定。
    • Prompt： 翻译提示词，需包含 ${sourceText} 占位符表示待翻译文本。
    • Stream： 是否流式返回，按需勾选。

关于 ${sourceText} 占位符

${sourceText} 是提示词模板中的占位符。每次翻译时，插件会把你在阅读器中划选的文本（即当前选中的上下文）填入这个位置，再将整段提示词发送给模型。可以把它理解为模板与实际原文之间的桥梁：模板写一次，原文每次动态替换。

使用时注意几点：

  • 必须保留： 模板中若缺少 ${sourceText}，模型就收不到待译原文，只会执行指令而无内容可译。

  • 放在末尾： 建议指令在前、${sourceText} 在后，让模型先读懂要求、再读到原文，符合其处理顺序。

  • 明确边界： 可用分隔符或标签把它包裹起来，避免模型把指令误当作待译内容。例如：

  • 只需一次： 占位符在模板中出现一次即可，无需重复。

借助这一机制，你可以在不改动原文选取方式的前提下，只通过调整提示词来改变模型的处理方式——例如换成「请总结下文要点」或「请润色下文」，配合多个 Custom GPT 槽位，就能让同一段选中文本得到不同的输出。

第五步：验证与使用

  1. 测试翻译： 打开一篇 PDF 文档，在阅读界面中划选文本或开启侧栏翻译，确认当前引擎能正常返回译文。
  2. 切换引擎对比： 在 Translation Services 下拉中切换 DeepLX(API) 与 Custom GPT 1，对比两种引擎的翻译效果，选择更适合的方案。
  3. 拼接翻译： 翻译跨行或跨段的内容时，可在条目面板勾选 Concat Mode（或按住 Ctrl/⌘）连续选取多段文本，一并翻译。
  4. 排查问题： 若返回异常，优先检查服务地址是否可达、Secret（API Key）是否正确、API 地址是否包含 /v1 等路径细节，以及 Prompt 是否保留了 ${sourceText} 占位符。

总结

完成以上配置后，Translate for Zotero 可以在同一插件中切换不同的翻译引擎：

  • 用 DeepLX 翻译 PDF 中的术语或段落。
  • 在 DeepLX 与多个 Custom GPT 槽位间切换，通过不同的提示词调整译文风格（如直译、润色、总结）。
  • 将译文一键 Add Translation to Note，保存为 Zotero 条目的笔记。

配置只需进行一次，之后即可在 Zotero 阅读器中直接使用。
